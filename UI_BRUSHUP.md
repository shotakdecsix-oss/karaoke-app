# カラオケAIオススメ UIブラッシュアップ設計書

対象: `index.html` のみ。`/api/recommend` と localStorage 構造は変更しない。
実装者向け（Sonnet想定）。モックアップ `ui-mockup.html` が見た目の正。

---

## 1. 現状の課題

1. **動線が縦一直線**: 毎回使う「気分・セッション入力→オススメ」と、初回だけ設定する「プロフィール・好きな曲」が同じ縦積みにあり、CTAボタンが画面中程に埋まる。
2. **結果が下に追加されるだけ**: オススメ実行後、結果までスクロールが必要。ロード中はテキストスピナーのみ。
3. **チップの見た目が2種類**: 削除できるタグ（.tag）と気分タグ（.mood-tag）でスタイルが微妙に不統一。
4. **カラオケ店（暗所・片手）での使用**が考慮されていない: CTAが固定でなく、タッチターゲットが小さい箇所がある。

## 2. デザイン方針

- ネオン系のピンク×紫（`#ff5c8a` × `#7c6cff`）のアイデンティティは維持。**グラデーションはCTAと選択状態のみ**に限定し、他はフラットに整理。
- 画面を「**毎回使うもの**（気分・セッション・結果）」と「**たまに設定するもの**（プロフィール・好きな曲・除外曲）」に分離。後者はヘッダー⚙から開く**設定シート**へ。
- CTAは**画面下部固定**。結果はCTAの上の領域に表示し、実行時に自動スクロール。
- ロード中は**スケルトンカード**を5枚表示（体感待ち時間の軽減。「10秒ほどかかります」文言は維持）。

## 3. レイアウト仕様

```
┌──────────────────────┐
│ 🎤 カラオケAIオススメ    [⚙] │ ← 設定シートを開く
├──────────────────────┤
│ 🌤 今日の気分                 │
│ [激しい][しっとり][ゆっくり]… │ ← 選択チップ（複数可）
│ [自由記入 textarea]           │
│ 🏟 今日のセッション（6時リセット）│
│  周りで歌われた曲 [input][＋] │
│  (タグ列)                    │
│  自分が歌った曲 [input][＋]  │
│  (タグ列)                    │
│ ── 結果エリア ──             │
│ ① 曲名 / アーティスト カード  │
│    理由 / キー / 難易度       │
│    [🎤歌った][💖好き][🚫除外] │
│ ②〜⑤ …                      │
│ [🔄 別の候補を出す]           │
├──────────────────────┤
│ [ オススメしてもらう 🎶 ]     │ ← 下部固定CTA（グラデ）
└──────────────────────┘
```

### 3.1 ヘッダー
- タイトル左寄せ、右端に⚙アイコンボタン（44px）。ビルド時刻表示は削除（設定シート最下部に移動）。
- ⚙にプロフィール未設定バッジ（生まれ年がデフォルトのままなら小さな赤ドット）。

### 3.2 設定シート（新規、下からスライドするボトムシート）
- 中身は現行の「👤プロフィール」「💖好きな曲・アーティスト」「🚫歌わない・知らない曲」をそのまま移設（ID・localStorageキー不変）。
- overlay タップ or ✕で閉じる。`transform: translateY` アニメーション 0.25s。
- シート高さは最大85vh、内部スクロール。

### 3.3 気分セクション
- 気分チップと自由記入は現行機能のまま。チップは44px高に拡大、選択時グラデ＋太字（現行踏襲）。
- チップとタグの視覚言語を統一: 未選択チップ＝`--card2`背景+境界線、選択＝グラデ。削除可能タグ＝`--card2`+✕ボタン。角丸は全て999px。

### 3.4 セッションセクション
- 現行のまま2入力＋タグ列。入力欄とボタンは高さ48pxに統一。

### 3.5 結果エリア
- 実行時: `results` へ `scrollIntoView({behavior:'smooth'})`。スケルトンカード5枚（タイトル行・アーティスト行・理由2行をシマー表示）。
- 曲カード: 左上に丸数字ランク（①…⑤の代わりに数字入り円形バッジ、グラデ背景）。曲名 1.05rem 太字、アーティストは`--accent2`。理由は行間1.6。
- アクション3ボタンは等幅・高さ44px・アイコン＋短ラベル（歌った / 好き / 除外）。押下後は✅状態（現行踏襲）。
- 「🔄 別の候補を出す」は結果の直下（現行踏襲）。
- エラー時: カード型のエラー表示＋「再試行」ボタン（statusテキストのみ→ボタン付きに）。

### 3.6 下部固定CTA
- `position:fixed; bottom:0`、グラデ背景、高さ56px＋safe-area。実行中は disabled＋スピナー内蔵（「AIが選曲中…」）。
- `main` に `padding-bottom: 96px` を確保。

## 4. デザイントークン

```css
:root {
  --bg:#0f0f1a; --card:#1c1c2e; --card2:#26263c; --line:#3a3a55;
  --accent:#ff5c8a; --accent2:#7c6cff;
  --grad:linear-gradient(90deg,var(--accent),var(--accent2));
  --text:#f0f0f5; --muted:#9a9ab0; --radius:14px;
}
```

- カラオケは暗所利用が主のためダーク固定でよい（ライトテーマ不要）。
- フォーカス: `:focus-visible { outline:2px solid var(--accent2); }`

## 5. 実装手順（推奨順）

1. ヘッダー＋⚙、プロフィール系3セクションをボトムシートへ移設（DOM移動のみ、JSのID参照は不変なので動くはず。開閉JSを追加）。
2. 下部固定CTA化＋結果への自動スクロール。
3. スケルトンローディング（CSSのみで実装、`@keyframes shimmer`）。
4. チップ/タグ/ボタンのスタイル統一、ランクバッジ、エラーカード。

## 6. 受け入れ基準

- [ ] オススメ取得・歌った/好き/除外・別の候補・タグ追加削除・プロフィール保存が従来通り動く
- [ ] localStorage のキーとデータ形式が変わっていない（既存ユーザーのデータが引き継がれる）
- [ ] CTAが常に画面下部に見えており、実行すると結果へ自動スクロールする
- [ ] ロード中にスケルトンカードが表示される
- [ ] プロフィール系は⚙から開くシートに移動し、メイン画面が「気分→セッション→結果」だけになっている
- [ ] 全ボタン・チップのタッチターゲットが44px以上
